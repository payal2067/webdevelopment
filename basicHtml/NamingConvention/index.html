Types of Naming Convention

  1. camelCase
    • In camelCase:
      • First word starts with lowercase
      • Next words start with capital letters
    • Example:
      • firstName
      • totalMarks
    • Use:
      • Variables and functions
  2. PascalCase
    • In PascalCase:
      • Every word starts with a capital letter
    • Example:
      • StudentName
      • TotalMarks
    • Use:
      • StudentData
  3. snake_case
    • In snake_case:
      • Words are separated by underscore (_)
    • Example:
      • student_name
      • user_login_status
    • Use:
      • Column names
  4. kebab-case
    • In kebab-case:
      • Words are separated by hyphen (-)
    • Example:
      • product-list
      • user-profile
    • Use:
      • product-list
      • main-header
  5. UPPER_CASE
    • In UPPER_CASE:
      • All letters are capital
      • Words separated by underscore
    • Example:
      • MAX_USERS
      • API_KEY
    • Use:
      • MAX_USERS
  6. lowercase
    • In lowercase:
      • All letters are small
      • No spaces
    • Example:
      • index.html
      • script.js
    • Use:
      • google.com
  7. Hungarian notation(rare today)
      • This style adds a prefix describing the variable type.
    • Example:
      • strName
      • boolLogin
    • Use:
      • string
